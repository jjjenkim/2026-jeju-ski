# 🚀 Notion to Obsidian Organizer - 사용 가이드

## ✨ 새로운 기능: 증분 업데이트

이제 3가지 실행 옵션을 선택할 수 있습니다!

### 옵션 1: 기존 스냅샷 사용 ⚡
- **소요 시간**: 0초
- **언제 사용**: 리포트만 다시 보고 싶을 때
- **Notion API 호출**: 없음

### 옵션 2: 증분 업데이트 (추천) 🔄
- **소요 시간**: 5-10분
- **언제 사용**: 한 달간 문서를 추가/수정했을 때
- **작동 방식**:
  1. 기존 775개 페이지 데이터 로드
  2. Notion에서 전체 페이지 ID만 빠르게 스캔
  3. 새로운 페이지와 수정된 페이지만 찾기
  4. 변경된 페이지만 메타데이터 추출
  5. 기존 데이터와 병합
  6. 리포트 생성

**예시:**
```
기존 데이터: 775개
새 페이지: 50개
수정된 페이지: 25개
→ 75개만 처리 (5-10분)
→ 최종: 825개 페이지
```

### 옵션 3: 전체 새로 스캔 🔄
- **소요 시간**: 60-90분
- **언제 사용**: 처음 실행하거나 완전히 새로 시작할 때
- **Notion API 호출**: 전체 페이지 스캔

## 📊 시나리오별 사용법

### 시나리오 1: 매일 리포트 확인
```
실행 → 옵션 1 선택 → 0초 완료
```

### 시나리오 2: 한 달간 100개 문서 추가
```
실행 → 옵션 2 선택 → 5-10분 소요
→ 기존 775개 + 새 100개 = 875개
```

### 시나리오 3: 처음 사용
```
실행 → 옵션 3 자동 선택 → 60-90분 소요
→ 전체 스캔 후 스냅샷 저장
```

## 🎯 추천 사용 패턴

**주 1회 증분 업데이트:**
- 월요일: 옵션 2 (증분 업데이트)
- 화~금: 옵션 1 (스냅샷 사용)

**월 1회 전체 스캔:**
- 매월 1일: 옵션 3 (전체 새로고침)
- 나머지: 옵션 2 (증분 업데이트)

## 💡 팁

1. **스냅샷 파일 위치**: `output/snapshot_*.json`
2. **스냅샷 삭제하면**: 자동으로 전체 스캔
3. **증분 업데이트 실패하면**: 자동으로 전체 스캔으로 전환
4. **리포트 위치**: Obsidian 볼트의 `Notion_Index/` 폴더

## 🚀 실행 방법

```bash
cd Notion_Organizer

```
