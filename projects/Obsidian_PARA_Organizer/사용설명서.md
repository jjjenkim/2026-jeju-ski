# 🗂️ Obsidian PARA Organizer 사용 설명서

## 📋 개요

Obsidian 볼트의 파일들을 **PARA + 제텔카스텐** 구조로 자동 정리하는 프로그램입니다.

### 주요 기능
- ✅ PARA 폴더 구조 자동 생성
- ✅ 메타데이터 기반 파일 분류 (AI 없음, 토큰 소비 없음)
- ✅ 자동 파일 이동
- ✅ MOC (Map of Content) 자동 생성
- ✅ Dashboard 자동 생성
- ✅ 백업 자동 생성

---

## 🚀 빠른 시작

### 1. 설정 파일 수정

`config.txt` 파일을 열어 볼트 경로를 설정하세요:

```txt
VAULT_PATH=/Users/jenkim/Library/Mobile Documents/iCloud~md~obsidian/Documents/Mac book 옵시디언
```

### 2. 실행

**Mac:**
- `실행.command` 파일을 더블클릭

**터미널:**
```bash
cd Obsidian_PARA_Organizer
python3 para_organizer.py
```

### 3. 결과 확인

Obsidian에서 `PARA_System` 폴더를 확인하세요!

## 🏷️ 파일 분류 규칙

### Projects (100/)

**다음 중 하나라도 해당하면 Project로 분류:**
- `#project` 태그
- YAML: `type: project`
- YAML: `status: active/planning/completed`
- 키워드: "프로젝트", "마감", "deadline", "목표"

**예시:**
```markdown
---
type: project
status: active
deadline: 2026-02-01
---
# 웹사이트 리뉴얼
```

### Areas (200/)

**다음 태그로 분류:**
- `#work`, `#business` → Work
- `#personal`, `#life` → Personal
- `#health`, `#fitness` → Health
- `#finance`, `#money` → Finance

### Resources (300/)

#### Inbox
- 최근 7일 이내 생성
- 태그 없음

#### Literature Notes
- `#book`, `#article`, `#video`, `#paper` 태그
- YAML: `source:` 또는 `author:` 속성

**예시:**
```markdown
---
type: literature
source: "책 제목"
author: "저자"
---
# 독서 노트
```

#### Permanent Notes
- 링크가 5개 이상 (`[[링크]]`)

#### References
- `#reference` 태그
- 기본 분류

### Archive (400/)

- `#archive` 태그
- YAML: `status: archived`
- 90일 이상 수정 안 됨 + 완료 상태

## ⚙️ 설정 옵션

### config.txt

```txt
# 볼트 경로 (필수)
VAULT_PATH=/path/to/vault

# PARA 폴더 이름
PARA_ROOT=PARA_System

# 백업 생성 여부
CREATE_BACKUP=true

# 자동 이동 (false면 분류만)
AUTO_MOVE=true

# Inbox 기준 (일)
INBOX_DAYS=7

# Archive 기준 (일)
ARCHIVE_DAYS=90
```

## 💡 사용 팁

### 1. 태그 활용

파일에 적절한 태그를 추가하면 더 정확하게 분류됩니다:

```markdown
---
tags: [project, work]
status: active
---
# 내 프로젝트
```

### 2. YAML Frontmatter

파일 상단에 메타데이터를 추가하세요:

```markdown
---
type: literature
source: "책 제목"
author: "저자명"
tags: [book, AI]
---
```

### 3. 링크 활용

Permanent Notes는 링크가 많을수록 자동 분류됩니다:

```markdown
# 핵심 개념

[[관련 개념 1]]과 [[관련 개념 2]]는 [[기본 원리]]에서 파생됩니다.
```

### 4. 정기적 실행

- 주 1회: 새 파일 정리
- 월 1회: 전체 재정리

## 🔒 안전 기능

### 자동 백업
- 실행 전 자동 백업 생성
- `_backup_YYYYMMDD_HHMM/` 폴더

### 파일명 충돌 방지
- 동일 이름 파일: `filename_1.md`, `filename_2.md`

### 원본 보존
- PARA 폴더 외부 파일만 이동
- 숨김 파일 제외

## ❓ FAQ

### Q: 기존 폴더 구조는 어떻게 되나요?
A: PARA 폴더 외부의 파일만 이동합니다. 기존 구조는 유지됩니다.

### Q: 잘못 분류된 파일은?
A: 수동으로 이동하거나 태그를 수정 후 재실행하세요.

### Q: 백업은 어디에?
A: 볼트 루트의 `_backup_YYYYMMDD_HHMM/` 폴더에 저장됩니다.

### Q: 실행 취소는?
A: 백업 폴더에서 복원하거나 Obsidian의 파일 복구 기능을 사용하세요.

## 🆘 문제 해결

### 파일이 이동되지 않음
- `config.txt`의 `AUTO_MOVE=true` 확인
- 파일 권한 확인

### 분류가 이상함
- 태그 또는 YAML frontmatter 추가
- 수동으로 이동 후 재실행

### 오류 발생
- Python 3 설치 확인: `python3 --version`
- 볼트 경로 확인
- 파일 권한 확인
